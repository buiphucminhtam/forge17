## 1. Test Strategy

### 1.1 Scope

| Package | Scope |
|---------|-------|
| **forgenexus** | Code intelligence engine — agents, graph, scanner, RAG, CLI |
| **mcp** | Global MCP server — tools, pipeline, parsers, errors |

### 1.2 Goals

- Maintain ≥80% line coverage for both packages
- All new code requires tests before merge
- Integration tests cover end-to-end pipeline flows

### 1.3 Test Types

| Type | Location | Run In |
|------|----------|--------|
| Unit | `**/*.test.ts` | CI (every PR) |
| Integration | `tests/integration/` | CI (full) + manual |
| E2E | `tests/e2e/` | Manual (gate release) |

## 4. Test Execution Schedule

| Trigger | Action |
|---------|--------|
| Every PR | Run `npm run test` for changed package(s) |
| Before merge to main | Run full test suite + coverage |
| Weekly | Full integration test run with Docker services |
| Pre-release | E2E test suite |

## 5. CI Configuration

### 5.1 Wave 1 (Parallel)

- `lint-and-format` — ESLint + Prettier (mcp)
- `typecheck` — TypeScript build (mcp)
- `test` — Unit tests (mcp)
- `commitlint` — Commit message validation
- `forgenexus-lint` — ESLint + Prettier (forgenexus)
- `forgenexus-typecheck` — TypeScript build (forgenexus)
- `forgenexus-test` — Unit tests (forgenexus)

### 5.2 Wave 2 (After test)

- `test-coverage` — Coverage gate (mcp)
- `forgenexus-coverage` — Coverage gate (forgenexus)

## 6. Known Limitations

1. **Coverage reporter bug** — `minimatch` conflict with `@vitest/coverage-v8`. Coverage data is collected but reporter errors. Tests pass.
2. **CLI tests deferred** — CLI tools need stub/mock setup before unit testing.
3. **E2E tests pending** — Full end-to-end pipeline test not yet implemented.
