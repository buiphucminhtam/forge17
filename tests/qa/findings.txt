# QA Findings ‚Äî 2026-04-13

## Critical Issues

### 1. Coverage Reporter Bug ‚Äî `minimatch` Conflict (Severity: MEDIUM)
- **Location:** `forgenexus/vitest.config.ts` + `forgenexus/package.json`
- **Problem:** `@vitest/coverage-v8` depends on `test-exclude` which uses `minimatch`, but the `minimatch` version in the dependency tree is incompatible with the exported API.
- **Impact:** Coverage data is collected but cannot be reported. CI will fail at the coverage gate even though tests pass.
- **Root Cause:** npm overrides conflicting with vitest's expected `minimatch` version.
- **Fix Applied:** Removed `minimatch: ^9.0.5` override from `forgenexus/package.json` overrides.
- **Remaining:** Still broken ‚Äî need to investigate `test-exclude` version or use alternative coverage provider.
- **Recommendation:** Either (a) pin `test-exclude` to compatible version, (b) switch to `v8` provider with custom excludes, or (c) use `c8` instead.

## Medium Issues

### 2. MCP `parseFrontmatter` Not Exported (Severity: MEDIUM)
- **Location:** `mcp/src/parsers/skill-parser.ts`
- **Problem:** The `parseFrontmatter` function is used internally but not exported. Tests attempted to import it but it doesn't exist in the module's export surface.
- **Impact:** Cannot write targeted tests for frontmatter parsing.
- **Fix Applied:** Refactored tests to use only exported functions (`getAllSkills`, `getSharedProtocols`).
- **Recommendation:** Add `parseFrontmatter` to exports if it's a useful utility.

### 3. MCP `pipeline-manager` Tests Need State Cleanup (Severity: LOW)
- **Location:** `mcp/src/state/pipeline-manager.test.ts`
- **Problem:** Tests write to `.forgewright/pipeline-state.json` in the actual workspace. If tests crash, state file may be left behind.
- **Impact:** Could affect subsequent test runs or actual pipeline state.
- **Fix Applied:** Added `beforeEach`/`afterEach` cleanup to remove state file.
- **Recommendation:** Use temp directory or isolated test workspace root.
